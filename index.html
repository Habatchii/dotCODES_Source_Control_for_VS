Bracket Classes- Brackets are the least common denominator for all dotCODES operations. They create container 'space' for all other elements and toolsets within the environment. Brackets are used to tag commands and functions that are used by the console during code completion or execution. They also help establish boundaries between major device families and their specifications. Below are the seven brackets.

Repository Prompt Classes- Repository prompts distinguish the many different API schemes in which the console may use during completion or execution. A repository may be accessed, commits performed, peer notes exchanged and many other functions in which the API may allow.

Actor Classes- Actors are the special words or commands used to build operations in which the console will complete or execute. They are registered in the console and called by the bracket that it is encapsulated within. An actor may be a basic word or an array of string commands.

Necessary Classes- Necessaries are declarations that can be appended to a bracket without starting a new session or writing to the definitions classes. They may be used to bind two or more documents by transclusion. They may also create new prompts, actors and operations.

Definitions- Definitions are used to build class identity and generally announce all instances of a class in the open console. Definitions must be announced before a session in order to complete or execute functions.

Operator Classes- Operators perform any maintenance within the console. They typically are used by the console's syntax to write commands and operations along with actors and brackets. An example line using all three would be: [[T_ACT = "copy all"_"place*directory/subdirectory"_OPERA = "close console"]]

The above simply states: In the T bracket, use actors "copy all" and "place" to copy all files selected to the directory/subdirectory folder. The operator "close console is used to complete the session.

Normal Web Services Clasess (NWS)- Normal Web Services classes build anticipated normal connections between the console and any web enabled network. They are differentiated for developers seeking secure connections during sessions. The classes may be emphasized based upon their TLD, URL, Web Address, HTTP class or any other normal connection.
